# 📚 ชั้นหนังสือ (Book Shelf)

แอปจัดการกองดองหนังสือ — ถ่ายปก เก็บคลัง เช็คซื้อซ้ำ ติดตามการอ่าน คอลเลกชันแยกหมวด และมังกรเลี้ยงที่โตตามจำนวนหนังสือ ✨

เวอร์ชันนี้เป็นเว็บมาตรฐาน (Vite + React) พร้อมขึ้น GitHub แล้วเปิดผ่านอินเทอร์เน็ตด้วย GitHub Pages

---

## ⚠️ อ่านก่อน — ข้อจำกัดเวอร์ชันเว็บ

1. **ข้อมูลเก็บใน `localStorage` ของเบราว์เซอร์** → อยู่กับเครื่อง/เบราว์เซอร์นั้น ๆ **ไม่ซิงก์ข้ามเครื่องอัตโนมัติ**
   - ย้ายข้อมูลระหว่างเครื่องได้ด้วยปุ่ม **สำรอง → ส่งออก/นำเข้า** (.json) ในแอป
   - ถ้าต้องการซิงก์ข้ามเครื่องจริง ๆ ต้องต่อ backend (เช่น Supabase) — ดูหัวข้อท้ายสุด
2. **ฟีเจอร์ AI (สแกนปกอัตโนมัติ + เช็คเล่มต่อไป)** ต้องใส่ **Anthropic API key ของคุณเอง**
   - เปิดเมนู **สำรอง/ตั้งค่า** ในแอป → ช่อง “เชื่อม AI” → วาง key (`sk-ant-...`)
   - key เก็บในเครื่องนี้เท่านั้น (อย่าใช้บนเครื่องสาธารณะ) — รับ key ได้ที่ https://console.anthropic.com
   - ถ้าไม่ใส่ key ก็ยังใช้แอปได้ครบ แค่ต้องกรอกข้อมูลหนังสือเอง
3. **สแกนนิ้ว/Face ID (WebAuthn)** ใช้ได้เฉพาะบน **HTTPS** — GitHub Pages เป็น HTTPS อยู่แล้ว ✅ (แต่บางอุปกรณ์/เบราว์เซอร์อาจไม่รองรับ จะถอยมาใช้ PIN ให้)

---

## 🖥️ รันในเครื่องก่อน (ทดสอบ)

ต้องมี Node.js 18+ ก่อน (https://nodejs.org)

```bash
npm install
npm run dev
```

เปิดลิงก์ที่ขึ้นมา (เช่น http://localhost:5173)

## 🚀 ขึ้น GitHub แล้วเปิดผ่านเน็ต (GitHub Pages)

### วิธีที่แนะนำ — ดีพลอยอัตโนมัติด้วย GitHub Actions

1. สร้าง repo ใหม่บน GitHub (เช่นชื่อ `book-shelf`)
2. อัปโหลด/พุชไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น branch `main`
   ```bash
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/<ชื่อคุณ>/<ชื่อ-repo>.git
   git push -u origin main
   ```
3. ไปที่ repo บน GitHub → **Settings → Pages**
4. หัวข้อ **Build and deployment → Source** เลือก **GitHub Actions**
5. รอสักครู่ (ดูสถานะที่แท็บ **Actions**) พอเสร็จจะได้ลิงก์เว็บ เช่น
   `https://<ชื่อคุณ>.github.io/<ชื่อ-repo>/`

> ไฟล์ `.github/workflows/deploy.yml` จะ build แล้วดีพลอยให้เองทุกครั้งที่ push ขึ้น `main`

## 🔧 เปลี่ยนรุ่นโมเดล AI (ถ้าจำเป็น)

โค้ดเรียกใช้โมเดล `claude-sonnet-4-6` อยู่ใน `src/App.jsx` (ค้นคำว่า `model:`)
ถ้าบัญชี/คีย์ของคุณใช้ชื่อรุ่นอื่น ให้แก้ตรงนั้นได้เลย

---

## 🔭 อยากได้ซิงก์ข้ามเครื่อง/หลายผู้ใช้จริง ๆ

ต้องเพิ่ม backend:
- **Auth + ฐานข้อมูล:** เช่น Supabase (รองรับ passkey/biometric ผ่าน WebAuthn ฝั่งเซิร์ฟเวอร์)
- ย้าย `store` (ที่ตอนนี้ใช้ localStorage) ไปอ่าน/เขียนผ่าน API ของ backend
- ย้ายการเรียก Anthropic API ไปไว้ฝั่งเซิร์ฟเวอร์ เพื่อไม่ต้องเปิดคีย์ในเบราว์เซอร์

โครงสร้างโค้ดแยก `store` กับ `callClaude` ออกมาให้แล้ว เปลี่ยนสองจุดนี้จุดเดียวก็ต่อ backend ได้
